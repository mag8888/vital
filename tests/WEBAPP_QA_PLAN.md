# QA Test Strategy (WebApp) — Vital

## Цель
Проверить, что клиентская часть WebApp стабильно работает после серии UI/навигационных изменений: нижнее/боковое меню, оверлеи, избранное, поддержка, корзина, страницы вторичного уровня.

## Объект тестирования (scope)
- **UI навигации**: нижнее меню, боковое меню (drawer), оверлеи, заголовки, стрелка назад.
- **Ключевые разделы**: Домой (каталог на главной), О нас, Поддержка, Избранное, Амбассадоры/Партнёры.
- **Внутренние страницы**: Сертификаты, Акции, Контакты, Корзина и прочие, открываемые через `openSection()`.
- **Избранное**: отображение карточек, кнопка “В корзину”, иконка сердечка.
- **Регрессия**: отсутствие JS syntax errors/дублей, которые ранее ломали “все кнопки”.

## Не входит (out of scope) без E2E браузера
- Реальные клики в браузере, скролл/жесты, визуальная регрессия скриншотами
- Telegram WebApp окружение (haptics, initData, openLink) — проверяется интеграционно

## Стратегия: risk‑based
Приоритеты (по риску “сломать прод”):
1. **JS не парсится / runtime error** → полностью неработоспособные кнопки/вкладки.
2. **Навигация** → пользователь “застрял”/не может вернуться.
3. **Корзина/избранное** → бизнес‑функции (добавление/индикаторы/избранное).
4. **UI консистентность** → иконки, выравнивание, мелкие UX детали.

## Тестовые окружения
- Desktop: Chromium/Safari (минимум)
- Mobile: iOS Safari / Android Chrome (минимум)

## Чек‑лист (ручной, быстрый регресс)
### Навигация/оверлеи
- **Домой**: нижнее меню видно всегда, карточки товаров загружаются.
- **О нас / Поддержка / Избранное / Амбассадоры**: нижнее меню **видно**, заголовок **по центру**.
- **Партнёры**: нижнее меню **видно**, заголовок **по центру**, **стрелка назад есть**.
- **Сертификаты/Акции/Контакты**: нижнее меню **скрыто** (оверлей полноэкранный), **стрелка назад есть**.

### Боковое меню (drawer)
- Иконки **outline**, одинаковый стиль (stroke currentColor).
- Пункты работают: Каталог/Сертификаты/Акции/Партнеры/Контакты.

### Избранное
- Карточки: кнопка **под ценой**, **по центру**.
- Сердечко: добавить/убрать из избранного, без перезагрузки.
- “В корзину”: добавляет, бейдж корзины обновляется.

## Автотесты (обязательный минимум)
- Smoke: структура HTML, отсутствие emoji‑иконок, корректные ссылки/onclick.
- Логика: контракты `openSection/closeSection` (классы main‑section/no‑back).
- Статика: отсутствие дублей критичных функций, синтаксис JS.

## Дефект‑репорт (шаблон)
- Заголовок
- Окружение (устройство/браузер/версия, prod build hash)
- Шаги воспроизведения
- Факт / Ожидание
- Логи консоли (ошибки)
- Скрин/видео
- Severity (Blocker/Critical/Major/Minor)
